- 同一时刻只能做一件事，队列的缺点
    - 与js的运行方式相关   js task 是单线程语言，相对于多线程同一时刻 js只能有一个任务在执行

    - 优点
        - 简单好理解
    - 缺点
        - 低效率

    - 线程 thread
        - 代码执行的最小单元

    - 加入数据请求 fatch/ajax，如果单线程（任务），并且同步的话，会给我们的页面显示带来什么麻烦？
        1. 慢不存在   2s
        2. 滚动或在页面交互的时候，数据请求在发生，假如ajax同步的阻塞当前js单线程，scroll不会响应。
        3. 加入所有任务都是同步任务（顺序执行）
            - 很多耗时任务的话，页面会进入僵死状态。
            - 定时器， 单线程 = 同步（简单，易读） + 异步任务（复杂性，event loop）  前端
